解決 PATH Wrapper 遞迴問題的技術方案

2026-03-17 | OpenClaw 技術報告

🔍 問題背景

原始問題：無限遞迴

當使用 PATH wrapper (~/.openclaw/bin/) 時，RTK 內部的 Command::new() 會根據 PATH 解析，找到 wrapper 而非真實二進位檔，導致無限遞迴。

~/.openclaw/bin/git (wrapper) └─ exec rtk git "$@" └─ Command::new("git") ← 解析到 wrapper！ └─ 無限循環 💀

💡 解決方案

新增 path_utils.rs

  • 智能路徑搜尋模組
  • 自動跳過 wrapper 目錄
  • 支援環境變數擴展

修改 utils.rs

  • 替換 resolve_binary()
  • 使用新路徑搜尋邏輯
  • 保持向後兼容

⚙️ 核心技術實現

const DEFAULT_SKIP_PATTERNS: &[&str] = &[ ".openclaw/bin", ".rtk/bin", "rtk-wrapper", ]; pub fn find_real_binary(cmd: &str) -> String { // 跳過 wrapper 目錄，找到真實二進位檔 ... }

關鍵創新：在 PATH 搜尋中智能跳過 wrapper 目錄，直接定位真實二進位檔

🎯 技術優勢

  • 解決遞迴問題：從根本上解決 wrapper 導致的無限遞迴
  • 保持 PATH 彈性：不需要修改系統 PATH，只需跳過特定目錄
  • 可擴展性：透過 RTK_SKIP_PATH 環境變數可自定義跳過模式
  • 向後兼容：不影響現有使用方式，無 wrapper 時行為不變
  • 回退機制：若 PATH 找不到，自動回退到常見安裝位置
