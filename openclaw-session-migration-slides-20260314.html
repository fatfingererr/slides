Session ID 格式遷移實錄與踩坑紀錄

v2026.2.9 → v2026.3.13 2026-03-14

🔍 問題發現

  • Doctor 指令中斷：執行 openclaw doctor --fix 時拋出錯誤
  • 錯誤訊息：Invalid session ID: agent:main:discord:channel:...
  • Gateway 啟動報錯：重啟後日誌出現相同錯誤
  • Doctor 無法完成設定檔遷移（dm.policy 欄位搬移）

🎯 根因分析

新版 v2026.3.13 新增了 Session ID 驗證規則：

/^[a-z0-9][a-z0-9._-]{0,127}$/i
  • 此規則不允許冒號（:）
  • 僅接受英數字、點、底線、連字號
  • Session key 與 Session ID 被嚴格區分

📊 Session Key vs Session ID

Session Key

agent:main:discord:channel:1475331209713160242
  • sessions.json 中的鍵
  • 用於路由與索引
  • 可包含冒號

Session ID

58092d7b-787a-4129-bb61-f90bff85e6c8
  • UUID 格式
  • 對應 .jsonl 轉錄檔
  • 必須符合正規表達式

💥 實際資料問題

舊版行為：Session key 被誤存為 sessionId

{ "agent:main:main": { "sessionId": "agent:main:discord:channel:1475331209713160242" // ❌ 這是 session key，不是 UUID！ } }
  • agent:main:main 條目的 sessionId 欄位錯誤
  • 舊格式轉錄檔：agent:main:discord:channel:...jsonl
  • 新版無法解析含冒號的檔名

⚠️ 踩坑紀錄：Gateway 記憶體覆寫

關鍵教訓：必須先 stop 再修改！
  • 第一次嘗試：Gateway 運行中修改 → doctor 成功 → Gateway 自動重啟 → 記憶體覆寫磁碟 → 修復失效
  • 第二次嘗試：修改後 restart → 關閉時寫回舊狀態 → 再次失效
  • 第三次（成功）：stop → 等待停止 → 修改 → 啟動 → 修復持久化

🔧 正確修復流程

  1. 停止 Gateway：openclaw gateway stop
  2. 備份 sessions.json：建立 .bak 備份檔
  3. 修正 sessionId：替換為 UUID 格式
  4. 封存舊轉錄檔：將含冒號檔案搬至 archived/
  5. 啟動 Gateway：使用 launchctl bootstrap
  6. 驗證修復：執行 doctor --fix 確認無錯誤

✅ 檢查清單與遷移

Doctor 完成的設定遷移：

舊路徑

channels.discord.dm.policy
channels.discord.dm.allowFrom

新路徑

channels.discord.dmPolicy
channels.discord.allowFrom
  • 設定檔備份：openclaw.json.bak
  • 檢查腳本已準備（支援多 agent 排查）
