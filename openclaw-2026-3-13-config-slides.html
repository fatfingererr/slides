🗑️ 移除的設定

1. DM Policy 欄位遷移

// 舊版 (2026.2.9) - 巢狀結構
"channels.discord.dm.policy": "allowlist"
"channels.discord.dm.allowFrom": ["USER_ID"]

已移除

2026.3.13 的 openclaw doctor --fix 自動將巢狀的 dm.policy 和 dm.allowFrom 提升到 channels.discord 層級。這是設定結構遷移，不是功能變更。

✨ 新增設定 (1/3)

channels.discord.dmPolicy

模式 說明
pairing 未知發送者會收到一次性配對碼，bot 不會處理其訊息
allowlist 僅允許 allowFrom 清單中的使用者（預設）
open 允許所有人（需搭配 allowFrom: ["*"]）
disabled 忽略所有 DM
"dmPolicy": "allowlist"
"allowFrom": ["USER_ID"]

✨ 新增設定 (2/3)

channels.discord.streaming

控制 Discord 回覆的串流模式

模式 說明
off 等待完整回覆後才發送（預設）
partial 即時串流部分回覆
progress 在 Discord 上等同 partial

💡 建議

設為 off 意味著 bot 回覆會等全部生成完成才一次發出，使用者需等待較長時間但回覆較完整穩定。

✨ 新增設定 (3/3)

執行核准與安全設定

"execApprovals": {
  "enabled": true,
  "target": "channel"
}
"tools.exec.ask": "off"
"tools.exec.security": "full"
  • execApprovals：啟用執行核准機制，核准請求發送到 Discord 頻道讓使用者確認
  • exec.ask：控制是否在執行工具呼叫前要求使用者確認（off = 不詢問）
  • exec.security：控制工具執行的安全等級（full = 完全信任）

🌐 瀏覽器工具設定

"browser.enabled": true,
"browser.defaultProfile": "user"
欄位 值 說明
enabled true 啟用瀏覽器工具讓 agent 在 session 中使用
defaultProfile user 使用登入的 Chrome 使用者設定檔（包含 cookies、登入狀態）

⚠️ 注意

2026.3.13 新增 Chrome DevTools MCP attach 模式，支援已登入的瀏覽器 session。Agent 可以使用你已登入的瀏覽器狀態。

🔄 變更的設定

1. 模型 Fallback 順序重組

// 舊版 (2026.2.9)
fallbacks: ["kimi-coding/k2p5", "openai-codex/gpt-5.2-codex", "anthropic/claude-opus-4-6"]

// 新版 (2026.3.13)
fallbacks: ["anthropic/claude-opus-4-6", "openai-codex/gpt-5.3-codex"]

已變更

  • 移除重複的 kimi-coding/k2p5（已是 primary）
  • Claude Opus 從第三順位提升到第一 fallback
  • GPT Codex 從 5.2 升級到 5.3

🔄 Session 重設模式變更

// 舊版 (2026.2.9)
"session.reset.mode": "daily",
"session.reset.idleMinutes": 120

// 新版 (2026.3.13)
"session.reset.mode": "idle",
"session.reset.idleMinutes": 99999999
項目 舊值 新值
mode daily idle
idleMinutes 120 (2小時) 99999999 (~190年)

影響

實質上永不自動重設 session。Session 只能透過手動 /new 或 /reset 觸發重設。適合需要長期記憶的個人助理場景。

🔒 安全影響評估

變更項目 風險等級 說明
exec.ask: "off" ⚠️ 中 CLI 層級不再逐一確認，依賴 execApprovals 把關
exec.security: "full" ⚠️ 中 完全信任模式，agent 可執行任何操作
browser.enabled + profile=user ⚠️ 中 Agent 可存取已登入的瀏覽器狀態
session.idleMinutes: 99999999 ✓ 低 長期 session 增加 token 消耗，無直接安全風險
dmPolicy: allowlist ✓ 安全 僅允許白名單使用者 DM
execApprovals.enabled: true ✓ 安全 執行需經 Discord 頻道核准

📝 總結與建議

整體評估

此設定適合個人使用、信任環境。exec.security=full + exec.ask=off 的組合移除了 CLI 層級的安全防線，但 execApprovals 在 Discord 層級提供了替代保護。

不建議在多使用者或公開環境使用此設定

  • 如需更高安全性，建議將 exec.ask 設為 "always"
  • 搭配 security: "deny" 使用白名單機制
  • 定期檢查 execApprovals 的核准記錄
