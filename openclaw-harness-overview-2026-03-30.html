🦞 OpenClaw 是什麼？Harness 又是什麼？

OpenClaw = AI Agent 運行框架

LLM 是大腦，OpenClaw 是身體。負責感知（Discord / 郵件 / RSS）、記憶（L0–L3 + LCM）、行動（165 Skills + Tools）和演化（Evolver）。

Harness = Agent 的運行環境

  • Session 管理 — 對話生命週期、隔離、並行
  • Tool 調度 — exec / read / write / browser / message
  • Context 管理 — softTrim / hardClear / LCM compaction
  • Memory — 4 層分級 + 語義搜尋 + temporal decay
  • Skill Routing — description match → SKILL.md → workflow
  • Cron Scheduler — 42 jobs 自動化編排

⏰ Cron Scheduler — 自動化引擎

42 Total 30 Enabled 12 Disabled

三種排程格式

  • cron expr — "0 8 * * *" 標準 crontab
  • every — 毫秒間隔 (e.g. 6000000=100min)
  • at — ISO timestamp 一次性執行

兩種 Payload

PayloadSession適用
systemEventMain需要 context 連續性
agentTurnIsolated獨立、不影響主 session

關鍵限制

  • systemEvent 只能 在 main session 執行
  • agentTurn 只能 在 isolated session 執行
  • 混用會導致 context 污染或 session 衝突

WakeMode

  • now — 立即觸發
  • next-heartbeat — 等下一次心跳

Delivery

  • none — 靜默執行
  • announce — 結果發到 Discord
  • webhook — POST 到外部 URL

Temporal Decay 公式

score_final = score × 0.5^(age / halfLife)

SOUL 冷熱分離

Context 保護規則

  • ❌ 禁止 cat large-file
  • ❌ 禁止 >10KB 輸出到 context
  • ✅ ls -lh 先檢查大小
  • ✅ jq 'keys' 查看結構
  • ✅ wc -l file && head -20
  • ✅ sessions_spawn 大型分析

⚡ 原則：保護 context window 就是保護思考品質。大檔案用 sub-agent 隔離處理。

🧬 Evolver — 自我演化引擎

4-Stage Pipeline（23:00–23:15）

🧠 Stage 1 — Brain (23:00)

分析 session 記錄，識別模式與問題，產生 GEP (Guided Evolution Prompt)

✋ Stage 2 — Hand (23:05)

解析 GEP 中的修改指令，對 workspace 檔案執行變更

🔒 Stage 3 — Solidify (23:10)

驗證修改結果正確性，確保不破壞現有功能

📢 Stage 4 — Notify (23:15)

Discord 通知演化結果，摘要今日變更

安全約束

  • 只修改 workspaces/personal/
  • ❌ 不動 SOUL.md / system prompt
  • ❌ 不動 security config
  • ❌ 不刪 git-tracked 檔案
  • Strategy: repair-only

GEP 結構

Guided Evolution Prompt
├── 觀察 (Observations)
│   └── 從 session 記錄提取
├── 問題 (Problems)
│   └── 重複錯誤、效率瓶頸
├── 建議 (Suggestions)
│   └── 具體修改指令
└── 驗證 (Verification)
    └── 預期結果 + 測試方式

⚡ Agent 每天反省自己的行為，自動修正，持續進化。不是 AGI，是 ACI (Autonomous Continuous Improvement)。
